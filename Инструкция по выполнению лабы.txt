 («Студент-практикант») создайте WPF-приложение с использованием Entity Framework Core и подхода Code First. Определите три сущности: Student (свойства: Id, FirstName, LastName, DateOfBirth, Email, Password, University, Specialty), Company (Id, Name, Industry, City) и Report (Id, SubmissionDate, Topic, Grade, StudentId, CompanyId). В классе контекста AppDbContext, унаследованном от DbContext, объявите DbSet<Student>, DbSet<Company>, DbSet<Report>. В методе OnModelCreating настройте модель исключительно через Fluent API: задайте первичные ключи с HasKey() и автоинкремент через ValueGeneratedOnAdd(), настройте обязательные поля с IsRequired(), ограничьте длину строк через HasMaxLength(), создайте уникальные индексы для Email через HasIndex().IsUnique(). Реализуйте связи: «Студент → одна Компания» (HasOne/WithMany) и «Студент → много Отчётов» (HasOne/WithMany) с указанием внешних ключей через HasForeignKey() и каскадным удалением через OnDelete(DeleteBehavior.Cascade). Для CRUD-операций используйте методы Add(), ToList(), Remove() и SaveChanges(). Создайте миграции через Package Manager Console: Add-Migration InitialCreate, затем Update-Database; при изменении модели добавляйте новые миграции. Реализуйте упрощённую ролевую модель (например, выбор роли при запуске) с разным отображением функционала. В отчёте оформите: титульный лист по шаблону, описание архитектуры, примеры кода Fluent API, скриншоты интерфейса для обеих ролей, список команд миграций. Убедитесь, что в коде нет атрибутов данных ([Key], [Required]) — вся конфигурация только через Fluent API.
